v0.11 Recommendations

NOT YET SCOPED
1
Supabase-backed run/approval stores as default — promote RUDHRA_RUN_STORE_BACKEND=database and RUDHRA_APPROVAL_STORE_BACKEND=database as the recommended production default. JSONL was the right default at Stage 3; the platform has outgrown it for multi-node deployments.
›
2
OTel metrics exporter — add a Prometheus or OTLP metrics exporter alongside spans. Key metrics: run_duration_histogram, tool_call_count, token_input_total, token_output_total, approval_pending_gauge. Enables SLA alerting without log parsing.
›
3
Async evals run — POST /studio/v1/evals/run/async returning a job ID with polling via GET .../status. Large scenario packs (50+ cases) are not practical with the current synchronous call.
›
4
Monaco editor in spec editor — replace the plain textarea with Monaco (already a transitive Angular dependency via VS Code extensions). Schema-aware autocomplete and inline YAML linting would significantly reduce save failures.
›
5
Agent spec versioning — add a version field to the agent spec and a GET /studio/v1/agents/{name}/spec/history endpoint. Operators currently have no rollback path after a bad save.
›
